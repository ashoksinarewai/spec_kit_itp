# SpecDriven

Spec-driven development workspace using specifications, constitution, and Cursor/Spec-Kit workflows.

## Structure

- **`.specify/`** — Spec-Kit configuration: memory (constitution), scripts, and templates for specs, plans, tasks, and checklists.
- **`.cursor/`** — Cursor commands for Spec-Kit flows (specify, plan, tasks, implement, etc.).

## Getting Started

1. Use the Cursor commands (e.g. `speckit.specify`, `speckit.plan`, `speckit.tasks`) to create and refine specs and plans.
2. Follow the constitution in `.specify/memory/constitution.md` for project principles and constraints.
3. Implement from the generated specs and task breakdowns.
